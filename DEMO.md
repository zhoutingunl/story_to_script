# Demo 演示脚本（5–8 分钟）

目标：展示「一篇小说 → 结构化剧本」的完整链路，以及两个创新点与"反造假"设计。

## 0. 准备（30 秒）

```bash
source .venv/bin/activate
STORY2SCRIPT_AI_BACKEND=offline python -m story2script.app
# 打开 http://127.0.0.1:8000
```

> 用离线模式演示：无需 VPN、瞬时出结果、每次一致。fixture 是真实 MiniMax-M3 输出录制而来。

## 1. 开场与痛点（1 分钟）

- 一句话立意：**小说作者有故事、缺剧本经验**；本工具把连续叙事变成场景驱动的剧本初稿。
- 首页讲清流程：导入 → 章节解析 → 人物 → 场景 → 对白 → YAML，并附关系图与质量报告。

## 2. 生成剧本（1 分钟）

- 点 **「用示例小说生成」**（明代桐城穿越短篇，9 章 / 2.7 万字）。
- 瞬时跳到剧本详情页。强调：这条链路在 `hermes` 模式下能处理**任意上传**的新小说。

## 3. 剧本详情页（2–3 分钟）——重点

- **顶部指标**：32 场景 / 20 角色 / 313 句台词 / 对白密度 0.53 / 281 原文提取 + 32 AI 扩写。
- **人物关系图（创新点一）**：力导向图，庞雨（主角，橙色大点）居中；点节点高亮其关系（主仆/亲属/敌对/退婚…）。强调纯 SVG、零外部依赖。
- **质量报告（创新点二）**：场景过长 / 对白过少 / 角色失衡(Gini 0.69) / 原文覆盖率 1.0——都是从结构算出的真实指标。
- **剧本正文**：按章展示场景。指出每场的 slugline、原文段落溯源；每句台词的 `原文`/`AI扩写` 徽章。
- **反造假**（点睛）：`mode` 不是模型说了算——程序在原文上复核，谎标会被纠正；场景可回查原文段落。

## 4. 导出与 Schema（1 分钟）

- 点 **「下载 script.yaml」**，打开看结构：`meta / characters / scenes(heading,source,elements) / story_graph / quality_report`。
- 翻到 [`yaml_schema.md`](yaml_schema.md)：为什么用 YAML、为什么 `source` 溯源、为什么 `mode` 标注——同时服务"可编辑"和"反造假"。

## 5. Dashboard 与工程（1 分钟）

- `/dashboard`：真实运行统计 + 埋点事件。
- 一句话工程总结：可插拔 AI + 离线回放（无网也能跑）、77 个单测、90% 覆盖率、9 个分模块 PR 的迭代历史。

## 收尾（30 秒）

- 取舍：只做扎实的核心链路 + 两个创新点，而非堆砌半成品。
- 商业化：个人版 / 专业版 / 企业版（网文·影视·动画）/ API 版。
